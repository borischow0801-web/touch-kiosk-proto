# Database Compatibility Skill

This project uses:

- Development database: MySQL 8
- Production database: HighGo database

Design for compatibility.

## Hard Rules

1. Primary key fields must be varchar(36).
2. IDs are generated by backend UUID, not database auto-increment.
3. Do not use database enum types.
4. Status fields use varchar.
5. Boolean values use tinyint/smallint with 0/1.
6. Store JSON as text, not database JSON type.
7. Avoid MySQL-specific functions in SQL.
8. Avoid stored procedures and triggers.
9. Use created_at, updated_at, deleted_at.
10. Prefer logical delete instead of physical delete.

## Naming Rules

- Table names: lower_snake_case
- Column names: lower_snake_case
- Index names: idx_table_column
- Unique names: uk_table_column

## Review Checklist

Before finalizing DB code:

- Check that no auto-increment is used.
- Check that no enum type is used.
- Check that no JSON database-specific type is used.
- Check that timestamps and logical delete are present.
- Check indexes for common query fields.
