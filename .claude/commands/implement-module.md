# Implement Module

$ARGUMENTS

1. 先扫描与该模块相关的已有代码。
2. 输出简短实施计划。
3. 只修改与本模块直接相关的文件。
4. 遵守项目 CLAUDE.md 和相关 skills。

6. 前端模块需包含 route、api、page、components。
7. 修改后运行可用的类型检查、构建或测试。
8. 最后输出：
   - 改动文件
   - 新增接口
   - 验证命令
