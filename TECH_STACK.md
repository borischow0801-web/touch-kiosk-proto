# 触摸屏政务查询原型（前端+后端）技术栈文档

> 目标：用 Cursor 快速开发一套“可点选式、零键盘输入”的触摸查询原型，部署到服务器，通过安卓/Win11 触摸屏浏览器访问演示效果。  
> 终端：1080×1920 竖屏；网络：政务外网；不需要扫码/打印。

---

## 1. 项目目标与约束

### 1.1 目标
- 竖屏触摸友好：**所有必须点击的控件集中在屏幕中下部**
- **零键盘输入**：用户只点击页面元素，不调用系统软键盘
- 原型优先：能快速上线演示，便于迭代
- 可扩展：后续可接入真实事项库/部门数据、统计、内容配置后台等

### 1.2 约束
- 政务外网环境：**不依赖公网 CDN**（字体/图标/JS/CSS 必须本地化）
- 设备双系统：安卓 / Win11（原型先浏览器访问，后续再 Kiosk 化）
- 分辨率：1080×1920（portrait）

---

## 2. 总体架构

```
触摸屏浏览器（安卓 Chrome / Win11 Edge）
        |
        | HTTP/HTTPS（内网）
        v
Nginx（静态资源 + 反向代理）
        |
        +--> 前端 SPA 静态文件（/）
        |
        +--> 后端 REST API（/api）
                     |
                     +--> SQLite（原型）/ PostgreSQL（后期）
                     +--> Redis（可选：缓存/配置/限流）
```

---

## 3. 最小功能清单（MVP）

1) **首页**：6 个常用事项大按钮（可配置）  
2) **按部门查**：部门列表 → 事项列表 → 事项详情  
3) **按主题查**：主题列表 → 事项列表 → 事项详情  
4) **底部常驻导航**：首页 / 返回 / 重来 / 帮助  
5) **无操作自动回首页**：90 秒（可配置）  
6) **全程不出现系统键盘**：不使用 input；或 input 必须 readonly 且不触发焦点

## 4. 前端技术栈（推荐）

### 4.1 核心框架
- Vue 3
- TypeScript
- Vite

### 4.2 工程与依赖
- Vue Router（页面路由）
- Pinia（状态管理，轻量够用）
- Tailwind CSS（快速出 UI）
- HTTP：fetch 或 axios（任选其一）

### 4.3 UI/交互规范（必须写进开发约束）
#### 4.3.1 可触达区域分区（按竖屏 1080×1920）
- 顶部展示区：约 **0–29vh**（尽量不可点或极少点）
- 主交互区：约 **29vh–97vh**（所有按钮/列表/详情交互在此）
- 底部常驻导航：约 **3vh**（或固定 60px）

> 原则：任何“必须点”的按钮，不允许出现在 29vh 以上。

#### 4.3.2 触控尺寸（像素级建议）
- 最小可点击区域：≥ 120×120px
- 主按钮高度：≥ 140px（建议 160px）
- 列表卡片高度：≥ 160px
- 控件间距：≥ 20px
- 字体建议：
  - 标题：48–56px
  - 二级：36–44px
  - 正文：30–36px
  - 小字提示：不低于 26px

#### 4.3.3 零键盘输入策略
- 原型阶段：避免 `<input>`，用以下方式替代：
  - Chips 筛选（高频/个人/法人/社保/医保等）
  - A-Z / 拼音首字母侧栏（只点选）
  - 分页 + 大卡片列表
- 若必须出现输入框：使用 `readonly` + 自定义选择弹层，禁止系统键盘

#### 4.3.4 空闲回首页（前端实现）
- 全局 Idle Timer：90 秒无操作 -> `router.push('/home')`
- 触摸、点击、滚动均视为“操作”

#### 4.3.5 断网兜底（建议）
- API 超时/断网时显示“网络异常，请稍后”，并自动重试（如每 5 秒）
- 首页骨架可本地缓存（可选增强）

## 5. 后端技术栈（推荐：原型快 + 可演进）

### 5.1 核心框架
- Node.js（LTS）
- TypeScript
- Fastify（轻量快速）

### 5.2 数据存储分阶段
- P0 原型：SQLite（单文件）或 `data/*.json`
- P1 试点：PostgreSQL（审计友好、标准化）
- 可选：Redis（热点缓存、配置发布、基础限流）

### 5.3 API 设计（原型够用，REST）
- `GET /api/health` 健康检查
- `GET /api/config` UI 配置（首页常用事项、Idle 时间等）
- `GET /api/depts` 部门列表（可 `?hot=1`、`?letter=A`）
- `GET /api/topics` 主题列表（可 `?hot=1`）
- `GET /api/items` 事项列表（`?deptId=...&topicId=...&hot=1&page=1&pageSize=20`）
- `GET /api/items/:id` 事项详情
- `POST /api/metrics/click`（可选）点击统计

---

## 6. 部署技术栈（Ubuntu Server）

- Nginx：前端静态 + 反向代理 `/api`
- Docker Compose（推荐）或 PM2/systemd（原型可用）
- 政务外网注意：资源本地化、尽量 HTTPS、基础限流与日志落盘
