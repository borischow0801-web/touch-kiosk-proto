# 群众端首页模块配置收敛规则

## 一、结论

群众端首页不采用“管理端自由搭建页面”的模式。

首页 UI 采用固定设计槽位，管理端只负责控制固定模块的展示、排序和内容绑定。

## 二、原因

竖屏触摸大屏首页对视觉完整性要求较高。

如果管理端可以自由新增任意模块、任意名称、任意布局，会导致：

- 首页卡片比例失控
- 模块数量不可控
- 文案长度破坏版式
- 视觉风格不一致
- 新模块缺少对应路由、接口和图标
- 设计稿无法稳定落地

因此首页模块配置必须从“自由配置”调整为“固定槽位运营配置”。

## 三、固定模块槽位

一期首页固定支持以下模块槽位：

- 高频事项
- 按部门查
- 按主题查
- 政策公开
- 窗口导航
- 常见问题
- 通知公告
- 模范先锋岗

后续如需新增模块，必须先完成：

- 前端视觉设计
- 前端路由
- Public API
- 后端数据来源
- 管理端配置入口
- 测试与验收

不得仅通过管理端新增一个 moduleCode 就让群众端直接展示。

## 四、管理端允许配置的内容

管理端允许配置：

- 是否显示
- 排序
- 显示名称
- 副标题
- 绑定内容分类
- 绑定业务入口
- 是否首页推荐
- 是否高频

## 五、管理端禁止配置的内容

管理端禁止配置：

- 任意新增未知 moduleCode
- 任意新增未知 targetType
- 自定义卡片 CSS
- 自定义布局比例
- 自定义前端路由
- 自定义外部接口地址
- 自定义脚本
- 自定义 HTML

## 六、前端处理原则

群众端只渲染白名单模块。

未知模块：

- 不展示
- 不报错
- 不影响其他模块
- 可在前端日志中记录 warning

模块数量不足时：

- 不伪造远程数据
- 使用既有模块自适应布局
- 离线模式使用本地完整兜底模块

## 七、推荐实现方式

前端维护固定模块注册表：

- moduleCode
- 默认标题
- 默认副标题
- 默认图标
- 默认颜色
- 默认布局槽位
- 默认路由或目标类型

管理端保存配置时，只能从固定模块注册表中选择。

后端 Public API 只返回已发布、可见、白名单内的模块。

## 八、验收标准

- 管理端不能创建未知模块
- 群众端不会展示未知模块
- 首页在 6 至 8 个核心模块下视觉稳定
- 首页在 2 至 3 个模块下不出现布局崩坏
- 新增业务模块必须经过开发流程，不得只靠配置上线
