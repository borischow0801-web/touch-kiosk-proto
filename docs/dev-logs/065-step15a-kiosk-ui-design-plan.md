## 一、设计目标与范围

### 1.1 目标

为 **1080×1920 竖屏触摸大屏**（Android APK，Capacitor）提供统一、可落地的群众端视觉与交互规范，覆盖首页至三级详情、宣传展示与异常兜底，并与现有 Public API、`home_module` 配置能力对齐。

### 1.2 设备与使用场景

| 项 | 规范 |
|---|---|
| 分辨率 | 1080 × 1920（竖屏） |
| 用户 | 匿名公众，含老年用户 |
| 安装 | 壁挂高位触摸屏，**顶部不宜作为主要触控区** |
| 交互 | **仅点击/触摸**，无键盘、无输入框、无搜索框 |
| 会话 | **90 秒**无操作回首页并清理临时状态 |
| 网络 | 断网 / 503 / 接口失败 **不得白屏** |

### 1.3 必须覆盖的页面

| 层级 | 页面 | 当前路由/数据来源（参考现状） |
|---|---|---|
| L1 | 首页 | `/home` ← `GET /api/public/home/config` |
| L1 区块 | 高频事项 | 首页 `homeHotItems` → `/items/:itemId` |
| L2 | 按部门查 | `/depts` ← Public Service Guide |
| L2 | 按主题查 | `/topics` |
| L2 | 事项类型页 | `/item-types` |
| L2 | 事项列表页 | `/items` |
| L3 | 事项详情页 | `/items/:itemId` |
| L2 | 政策公开列表页 | `/content/policies`（及同类 content 模块） |
| L3 | 政策公开详情页 | `/content/policies/:id` |
| L2 | 窗口导航页 | 待开发 ← `GET /api/public/navigation/*` |
| L2 | 常见问题页 | `/content/faqs` |
| L2 | 通知公告页 | `/content/notices` |
| L2 | 模范先锋岗 / 宣传展示页 | 待开发 ← `GET /api/public/showcase/items` |
| 兜底 | 503 / 断网 / 接口失败 | 本地 `offlineHomeConfig` + 全页友好态 |
| L1 | 帮助页 | `/help` |

## 二、整体视觉关键词

| 关键词 | 释义 |
|---|---|
| **清爽舒适** | 大留白、低饱和、信息密度适中，减轻高位屏阅读压力 |
| **沿海城市气质** | 蓝天、白云、海平线、浅蓝白渐变，传达开放、可信、便民 |
| **幸福门意象** | 白色玻璃钢结构「大相框」抽象图形，框住海天一色，**非写实照片堆砌** |
| **Liquid Glass** | 轻玻璃拟态：半透明、背景模糊、细边框高光、柔和内阴影 |
| **错落有致的卡片** | 拒绝等大宫格；通过尺寸、列跨、纵向错位形成节奏 |
| **荣誉而不张扬** | 模范先锋岗：浅红/暖金点缀的横幅卡，**避免大面积正红** |
| **触摸优先** | 大触控目标、底部主导航、顶部纯展示为主 |
| **克制动效** | 150–280ms 级过渡，无连续闪烁、无复杂粒子 |

**禁止**：国徽、红头模板化政务风、密集小字表格墙、输入框/搜索框、系统键盘。

## 三、色彩方案

### 3.1 主色板（建议写入 `themeJson` 映射）

| 角色 | 色值（参考） | 用途 |
|---|---|---|
| 天空蓝 Sky | `#EAF4FF` → `#B8DCFF` 渐变 | 顶区背景、页面底色 |
| 海平线蓝 Horizon | `#5BA4E8` | 强调线、图标、链接态 |
| 深文本 Ink | `#1E3A5F` | 主标题、卡片标题 |
| 次文本 Mist | `#5C728A` | 副标题、说明、时间 |
| 玻璃面 Glass | `rgba(255,255,255,0.72)` | 卡片底、导航条 |
| 玻璃描边 Glass Border | `rgba(255,255,255,0.55)` | 1px 内高光边 |
| 阴影 Shadow | `rgba(30,80,140,0.12)` | 卡片外阴影，blur 16–24px |
| 暖金 Accent Gold | `#E8C872` | 模范先锋岗点缀、徽章描边 |
| 浅红 Honor | `#F5D6D0` / `#E8928A`（点缀） | 荣誉横幅底/小标签，**不作主背景** |
| 成功/可用 | `#3BA776` | 可选：窗口开放状态 |
| 警告/离线 | `#D4A017` | 离线提示条、503 条 |
| 不可用 | `#94A3B8` | 禁用、空状态辅助 |

### 3.2 渐变与背景

- **顶区（约 30% 屏高）**：自上而下 `#EAF4FF` → `#FFFFFF`，叠加 **低对比度** 云层 SVG/位图（opacity 0.15–0.25）。
- **中部操作区**：浅白 `#F7FBFF`，可带极淡水平波纹纹理（opacity ≤ 0.08）。
- **底部导航区**：玻璃条 `rgba(255,255,255,0.88)` + 顶部分割线 `rgba(91,164,232,0.2)`。

### 3.3 与 `themeJson` 的建议键名（开发期映射，非接口变更）

| themeJson 键 | 映射 |
|---|---|
| `primary` | Horizon 蓝 |
| `backgroundTop` | 天空渐变起止 |
| `glassOpacity` | 玻璃面透明度 |
| `accentGold` | 暖金 |
| `honorTint` | 浅红荣誉 tint |

> 本期 **不修改** api-spec；上表供后续实现时约定 **可选** 扩展字段，缺省走设计令牌默认值。

## 四、字体与字号层级

**字体栈（Android）**：`"HarmonyOS Sans SC", "Noto Sans SC", "Source Han Sans SC", sans-serif`（Implement 阶段打包或系统 fallback，**本轮不新增依赖**）。

| 层级 | 字号（px） | 字重 | 行高 | 用途 |
|---|---|---|---|---|
| H0 页面主标题 | 48–52 | 700 | 1.2 | 首页顶区主标题（`title`） |
| H1 区块标题 | 40–44 | 700 | 1.25 | 二级页顶栏标题、模块区标题 |
| H2 卡片标题 | 36–38 | 600 | 1.3 | 首页/列表大卡片主文案 |
| H3 列表标题 | 32–34 | 600 | 1.35 | 列表项标题 |
| Body 正文 | 28–30 | 400 | 1.5 | 详情正文、说明 |
| Caption 辅助 | 24–26 | 400 | 1.45 | 时间、科室、楼层、标签 |
| Nav 导航 | 28–30 | 600 | 1.2 | 底部四按钮 |
| Banner 滚动 | 26–28 | 500 | 1.4 | `bannerLines` 温馨提示 |

**原则**：最小可读正文 **不低于 24px**；关键操作卡片标题 **≥ 36px**；避免一段多层级混排超过 3 级。

## 五、卡片尺寸与触控尺寸

基准画布：**1080 × 1920**；安全边距：左右 **32px**，区块间距 **24px**。

### 5.1 触控目标

| 类型 | 最小尺寸 | 说明 |
|---|---|---|
| 主卡片（可点击） | **宽 ≥ 480px，高 ≥ 140px** | 首页模块、列表项 |
| 底部导航按钮 | **高 96px**，四等分 | 全宽固定底栏 |
| 高频事项胶囊 | 高 **140–160px**，全宽或半宽 | 首页热区 |
| 返回/筛选 Chip | 高 **72px**，宽 ≥ 120px | 列表筛选（无输入） |
| 卡片间距 | **≥ 20px** | 防误触 |

### 5.2 首页卡片尺寸档（错落布局）

| 档位 | 约尺寸（宽×高） | 用途 |
|---|---|---|
| XL | 1016 × 200 | 高频事项首条、模范先锋岗横幅 |
| L | 496 × 180 | 双列大卡 |
| M | 496 × 148 | 双列标准卡 |
| S | 320 × 132 | 侧列小卡、通知摘要 |
| Banner | 1016 × 88 | 荣誉/公告横条 |

> 双列基准：`(1080 - 32*2 - 24) / 2 = 496px` 单列宽。

## 六、首页布局方案

### 6.1 三区结构（与 architecture / kiosk-ui 一致）

```
┌────────────────────────────────────── 1080px ──────────────────────────────────────┐
│  顶区 ~29–30vh（约 560–576px）— 展示为主，少交互                                      │
│  ┌ 三色弧形标志 ┐  标题 title（H0）                    日期时间                        │
│  │   （用户素材）  │  副标题 subtitle（Caption）                                       │
│  └──────────────┘  bannerLines（1–2 行）                                             │
│  [ 幸福门抽象大相框 + 海天线 + 云层 — 装饰层，opacity 低 ]                              │
├──────────────────────────────────────────────────────────────────────────────────────┤
│  中区 ~61–62vh（约 1170–1200px）— 主操作区，可滚动                                      │
│  · 高频事项 homeHotItems（若有）                                                      │
│  · 模范先锋岗荣誉横幅（若有 showcase / 配置模块）                                        │
│  · 首页 modules 错落卡片区                                                            │
│  · noticeSummaries 通知摘要（若有）                                                   │
├──────────────────────────────────────────────────────────────────────────────────────┤
│  底栏 固定 ~96px + safe-area — 首页 | 返回 | 重来 | 帮助                               │
└──────────────────────────────────────────────────────────────────────────────────────┘
```

- **顶区**：禁止 `<input>`、`contenteditable`、可聚焦搜索；时钟只读展示。
- **中区**：所有业务入口；滚动条隐藏或极细；惯性滚动。
- **底栏**：`nav[]` 驱动；「返回」走 history，「重来」清 session 回 `/home?reset=1`。

### 6.2 顶区视觉组成

1. **左侧**：用户提供的 **三色弧形标志**（PNG/SVG 资源位，建议高 96–112px）。
2. **中部偏左**：`title` + `subtitle` + `bannerLines`（来自 Public API 或离线兜底）。
3. **右侧**：日期时间（现有逻辑保留，字号 Caption）。
4. **背景层**：幸福门 **抽象大相框**（白色玻璃钢结构线条）+ 海平线渐变 + 淡云；**不使用国徽**。

## 七、首页模块错落布局说明

### 7.1 设计原则

- **非等大宫格**：通过 `layoutType` + `sortOrder` 驱动卡片档位（见 §十三）。
- **视觉节奏**：左大右小、上大下双列、荣誉横幅全宽插入，避免「棋盘格」。
- **同一屏可见模块**：建议 **6–10** 个可见模块 + 热项；超出区域 **纵向滚动**。

### 7.2 推荐默认模块顺序（含位置互换）

用户要求：**「按部门查」与「政策公开」互换位置**。

在 **默认/离线** 与 **管理端未自定义排序** 时，推荐视觉顺序如下（`sortOrder` 由管理端发布配置，开发默认 seed 应对齐本顺序）：

| sortOrder | moduleCode（示例） | 显示名 | 布局档 | 说明 |
|---|---|---|---|---|
| 1 | `hot_items`（运行时区，非 module 表） | 高频事项 | XL 全宽 | 来自 `homeHotItems`，非 home_module |
| 2 | `showcase_pioneer` | 模范先锋岗 | Banner 全宽 | 浅红/暖金荣誉横幅，非大块红底 |
| 3 | `content_policies` | **政策公开** | L 左列 | **原「按部门查」位置，已上移** |
| 4 | `guide_topic` | 按主题查 | M 右列 | 与上行形成错落 |
| 5 | `guide_dept` | **按部门查** | L 左列 | **原「政策公开」附近位置，已下移** |
| 6 | `content_faqs` | 常见问题 | M 右列 | |
| 7 | `navigation` | 窗口导航 | M | route 或专用页 |
| 8 | `content_notices` | 通知公告 | S 或列表摘要 | 可与 API `noticeSummaries` 合并展示 |
| … | 其他 content 模块 | 解读/制度/目录等 | S/M 混排 | 按 `sortOrder` |

**明确声明**：相对 architecture 文档列举的默认顺序，本方案 **已将「按部门查」与「政策公开」在首页卡片区互换位置**；管理端发布时须通过 `home_module.sortOrder` 落实，离线兜底配置同步调整。

### 7.3 错落示意（ASCII）

```
[════════════ 高频事项 XL ════════════]

[══════ 模范先锋岗 荣誉 Banner ══════]

[ 政策公开 L  ]  [ 按主题查 M ]
[ 按部门查 L  ]  [ 常见问题 M ]
[ 窗口导航 M  ]  [ 通知公告 S ]
      ... 更多模块纵向滚动 ...
```

适用于：按部门查、按主题查、事项类型、事项列表、政策/FAQ/通知列表、窗口导航、宣传列表等。

```
┌─ 顶栏 120px ─────────────────────────────────────┐
│ ← 视觉返回区（可选装饰条）  H1 页面标题            │
├─ 筛选/面包 72–96px（可选）───────────────────────┤
│  Chip 筛选：部门/主题/楼层…（仅点击，无输入）        │
├─ 内容区 可滚动 ──────────────────────────────────┤
│  玻璃卡片列表 / 双列卡片 / 楼层-区域-窗口层级        │
├─ 底栏 固定 96px ─────────────────────────────────┤
│ 首页 | 返回 | 重来 | 帮助                          │
└──────────────────────────────────────────────────┘
```

| 元素 | 规范 |
|---|---|
| 顶栏 | 浅蓝渐变底 + 玻璃分隔；标题来自路由 meta 或 API |
| 筛选 | 横向滚动 Chip；选中态 Horizon 蓝描边 + 浅蓝底 |
| 列表 | 统一玻璃卡片；左图标/色条 + 标题 + 一行摘要 |
| 空状态 | 插画 +「暂无相关信息」+ 可点「返回首页」 |
| 加载 | 骨架屏（玻璃块 shimmer，**单次** 不超过 1.2s 循环） |
| 分页 | **大按钮**「上一页 / 下一页」或触底加载；**禁止**输入页码 |

## 九、三级详情页通用布局

适用于：事项详情、政策详情、通知详情、宣传详情。

```
┌─ 顶栏 H1 标题（可两行截断）──────────────────────┐
├─ Meta 条：发布时间 / 来源 / 科室（Caption）───────┤
├─ 正文区 可滚动 ──────────────────────────────────┤
│  · 分段标题（H2 级 32px）                          │
│  · 正文 Body 28–30px                               │
│  · 材料/流程：编号列表或时间轴（纯展示）             │
│  · 关联入口：「相关政策」「常见问题」→ 大按钮跳转    │
├─ 底栏 固定 ──────────────────────────────────────┤
```

**事项详情**（对齐 kiosk-ui skill）：基本信息、受理条件、材料清单、办理流程、办理地点/时间/时限、收费标准、依据、电话、关联政策/FAQ；字段空则 **「暂无相关信息」**。

**政策/通知详情**：标题 + 发布时间 + 正文 HTML/纯文本安全渲染（现有 sanitize 策略延续）；无下载强制输入。

## 十、列表页设计

### 10.1 办事指南列表（部门 / 主题 / 类型 / 事项）

| 页 | 列表单元 | 交互 |
|---|---|---|
| 按部门查 | 部门图标 + 部门名 + 可选 hot 角标 | 点击进入事项类型或事项列表 |
| 按主题查 | 主题图标 + 主题名 | 同上 |
| 事项类型 | 类型名 + 事项数量（若有） | → 事项列表 |
| 事项列表 | 事项名 + 所属部门/主题一行 | → 事项详情；底部分页大按钮 |

**筛选**：仅允许 Chip / 卡片点选（如热门、字母分组 **若后端提供分组 API**）；**禁止**拼音键盘输入。

### 10.2 政务公开列表（政策公开等）

- 卡片：标题 H3 + 发布日期 + 一行摘要。
- 政策公开列表页 `/content/policies` 视觉与 FAQ、通知一致，仅 icon/色条区分（政策：Horizon 蓝；FAQ：青绿；通知：暖金）。

### 10.3 窗口导航列表

- **一级**：楼层卡片（玻璃 + 楼层号大字）。
- **二级**：区域卡片。
- **三级**：窗口卡片（窗口号、业务名称、状态点）。
- 数据来源：`GET /api/public/navigation/floors|areas|windows`（实现阶段对接）。

### 10.4 宣传展示 / 模范先锋岗列表

- 列表为 **荣誉卡片**：左 thumbnail + 标题 + 类型标签（pioneer / model_worker）。
- 点击进入全屏或详情，顶区 **浅红 tint 横幅** + 正文，**禁止**全屏大红背景。

## 十二、异常兜底页设计

### 12.1 触发条件

| 场景 | 检测 | 用户可见 |
|---|---|---|
| 无已发布首页配置 | HTTP 503 / `code=503` | 离线首页 + 顶栏提示条 |
| 断网 / DNS 失败 | fetch 异常 | 同上或当前页内嵌提示 |
| 单页 API 失败 | 非 0 code / 5xx | 页内空状态，保留底栏 |
| 数据为空 | `list.length=0` | 「暂无相关信息」 |

### 12.2 503 / 离线首页

## 十三、动效方案

| 动效 | 参数 | 场景 |
|---|---|---|
| 页面进入 | opacity 0→1，translateY 12→0，**220ms**，ease-out | 路由切换 |
| 卡片按压 | scale 0.98，**120ms** | `:active` / touch |
| 卡片入场 |  stagger 40ms，opacity + translateY 8px，**180ms** | 首页模块加载 |
| 列表项 | 同上，最多 stagger **8** 项 | 避免长列表卡顿 |
| 骨架 shimmer | 1.2s 单次循环，低对比 | 加载 |
| 荣誉横幅 | 极慢 gradient shift 8s（optional，可关） | 模范先锋岗 |
| 禁止 | 无限 bounce、粒子、视差滚动、闪烁 | — |

**原则**：动效 **服务触摸反馈与层级感知**；尊重 `prefers-reduced-motion`（Android 可配置关闭动画）。

## 十四、home_module 配置字段与群众端 UI 映射建议

| 管理端字段 | 群众端 UI 表现 | 说明 |
|---|---|---|
| `moduleCode` | 稳定键；映射默认 icon、默认 layout 档位 | 如 `guide_dept`、`content_policies` |
| `moduleName` | 卡片主标题 H2 | 必填展示文案 |
| `moduleType` | 卡片形态族 | 如 `card` / `banner` / `hero` |
| `icon` | 卡片左侧/顶部图标名或 Public 文件 URL | 空则用 moduleCode 默认图标 |
| `color` | 卡片左侧 6px 色条或 icon tint | 空则 Horizon 蓝；荣誉模块可用 honorTint |
| `layoutType` | 错落尺寸档 | 建议枚举：`xl` `l` `m` `s` `banner` `full` |
| `targetType` | 路由行为 | `route` → 内部 path；`content` → `/content/{targetValue}` |
| `targetValue` | 路由参数 | 如 `/depts`、`policies` |
| `isVisible` | 是否渲染 | false 则不占位 |
| `sortOrder` | 错落网格排序 | 越小越靠上；**政策公开/按部门查互换靠此字段** |

### 14.1 `layoutType` 建议枚举（实现期约定）

| layoutType | 宽度列 | 高度档 | 典型 moduleCode |
|---|---|---|---|
| `banner` | 12/12 | 88–100px | 模范先锋岗 |
| `xl` | 12/12 | 200px | 首屏重点（若模块级热区） |
| `l` | 6/12 | 180px | 政策公开、按部门查 |
| `m` | 6/12 | 148px | 按主题查、FAQ |
| `s` | 4/12 或 6/12 | 132px | 通知摘要 |
| `full` | 12/12 | 160px | 全宽特殊入口 |

### 14.2 `themeJson` / Public `theme`

| theme 键 | UI 作用 |
|---|---|
| `primary` | 强调色、Chip 选中、链接 |
| `backgroundTop` | 顶区渐变 |
| `glassOpacity` | 卡片玻璃透明度 |
| `accentGold` / `honorTint` | 荣誉模块 |

**高频事项**、**noticeSummaries** 不在 `home_module` 表，由 Public API 组合区单独渲染，视觉档位参照 XL / S。
